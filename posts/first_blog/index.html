工作#

博客搭建#

首先说说博客搭建吧，我采用的方案是使用hugo+github pages来进行博客搭建。

hugo#

hugo官方是称自己是世界上用于构建网站最快的架构，并且给出了很多网站模版给用户使用，全部都是免费开源的模版，这点我是比较喜欢的。

对我搭建博客来说，它的功能就是将我的markdown文档转化为html文件，构建成为了静态网页的内容。

github pages#

github pages是github出的一个静态网页管理器，这个特性是非常适合用于管理hugo生成的静态网页的内容的，而且可以用户给出自定义域，那我的域名又可以用上了，非常舒服，touboli.site再次启动。

具体步骤#

安装hugo，在github上建一个username.github.io的库，使用cloudflare将域名进行处理

hugo操作#

hugo new site name

在当前目录中产生一个名字为name的文件夹，里面是hugo的相关文件夹。有些地方把这种操作成为创建站点。

cd name

进入name文件夹，可以进行进一步的工作了，文件夹的内容如下

其中主要用到的文件夹是content文件夹，写的文章都是放在这里content/post文件夹中；themes文件夹，使用的网页模版来自于themes文件夹，我主要使用hugo-PaperMod模版；public，存放hugo根据content中的文章生成的静态网页；hugo.toml，hugo的配置文件，官方文档在这里，可以进行自定义配置。

hugo new my-first-post.md

这会在content/posts 目录中创建该文件，在这个文件中初始会有title信息，date信息，draft信息，一般来说hugo在构建网站时不会发布草稿内容。

hugo

控制台输入hugo命令会转化你写的markdown文档为静态网页，生成的静态网页存放到public文件夹中。

hugo server -D

启动hugo的开发服务器来查看网站，在终端会显示URL。访问可以查看效果。

更多的有关内容可以查看hugo的文档

github pages操作#

点击仓库的settings，找到pages部分，将DNS部分添加为域名即可

结合hugo + github pages#

将hugo的public文件夹和创建的github pages关联起来，使用git完成

在public目录下执行以下命令

#在当前目录下新建代码库
git init

#连接远程仓库
git remote add origin github的仓库链接

#查看本地所有分支
git branch

#查看远程所有分支
git branch -a

#切换分支到main分支
git checkout main

#添加所有文件到暂存区
git add .

#提交暂存区到仓库区
git commit -m "提交备注"

#拉取远程仓库main分支内容与本地仓库内容进行同步
git pull origin main

#上传本地指定分支到远程仓库
git push origin main

图片插入#

在搭建好博客以后，插入图片成为了一个问题，在本地插入的图片转化为HTML网页以后会导致图片路径问题导致图片显示不出来，解决方法有两个：

一个是采用在线图床，使用图床链接来实现图片导入，优点就是非常方便，但缺点在于免费的图床不稳定，并且上传数量和容量都有一定限制，而付费的图床……那肯定是不考虑的。

另一个是使用本地图片，直接在markdown中引用本地图片，采用特殊的文件存放方式，经过网上查阅发现大致有两种方法。

在static目录下创建img子目录，把markdown要使用的图片放在static\img目录里。

在markdown文件里，按照如下格式引用图片(这里假设图片名称叫wechat.png)。这样最终public目录下生成的静态页面就可以引用到public\img下的图片了，使用以下的方式进行引用。

第二种本地图片插入方式（成功）#

然后就可以通过域名进行访问了，后续的升级扩展思路在于使用github actions来实现自动化部署，或者利用服务器进行静态网页的存放和展示。

思考#

在本周还有一件比较有意思的事情，或者说比较有争议的事情。

事情的情景如下，舍友三个人小假期出门旅游，因为担心高铁和火车人很多且抢不到票，所以他们在租车公司租车准备进行一个自驾游，相关的费用均摊，包括油费，高速费等等。但是不出意外，意外发生了，驾驶汽车的同学A在驾驶过程中闯红灯了，扣6分以及200元罚款（这个我也记不太清了，应该是这个处罚），驾照分是同学A自己承担，罚款的费用同学A是打算均摊，在收款的时候就此发生了分歧。

同学A是驾车同学，觉得罚款的费用需要均摊；同学B和同学A关系好，也不计较，表示可以；同学C认为这是同学A个人的责任，是同学A的驾驶技能不过关，费用应该个人承担，表示不支持均摊。就这件事情同学C来问宿舍成员的意见，其中就包括我的意见。经过思考以及和同学讨论以后，我觉得支付和不支付都有理由。

首先是支付角度，因为同学A属于是主动承担驾驶风险，并且没有额外的收益，大家的费用是均摊的，相当于同学A是免费当司机，这本身是不合理的，如果这次不进行均摊的话，寒了人家的心，以后谁还敢说我来当司机？在承担司机责任的时候总要掂量掂量，所以从这个角度来说分摊是比较合理的，可以看做是用罚款的钱来支付驾驶费用。

不支付的理由也很简单，驾驶员技能不过关造成的错误自己承担，并且这里还有一个额外因素，同学C是没有驾照的，在路上不能很好的进行安全驾驶提醒，也就是说不具备分摊责任的可能。

综上所述，其实是否均摊都各有各的理由，还是建议在开始之前就确定好是怎么处理此类事件，是多给司机角色更多的钱，自己担责呢？还是不给司机钱，分摊责任？应该事先确定好才是。

PS：据同学C的说辞，同学A开车是非常快且莽撞的，只是运气好，一直没有出事，因为我个人没有坐过他开的车，所以这里也不好确认，以上情景经过的模糊处理，但确实是真实发生在我的宿舍。
