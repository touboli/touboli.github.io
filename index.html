2024年5月第三周

首先说一下本周的工作 赤道仪项目目前进程 在赤道仪开发上面，现在已经实现能将Onstep程序烧录进去，并使用手机进行控制，这个是比较喜人的一个进步，问题也是比较奇葩，也就是有一个参数没有调整好，没有输出想要的电压，所以导致整个没有一点动静。 在周末晚上的操场上组好了装备进行设备的调整，整套装备也是非常的贵，有5000元左右，且这连入门都算不上。 整个装备结构采用塑料，在部分螺丝没有拧紧，以及外部有小风的情景下，最终的效果还是比较不错的，一般的标准是在0.6，图中可以达到总计1.28左右的水平，已经是很不错的水平了，如果进行金属结构的替换，并且找到一个比较好的观测点，那效果无疑会更好。 项目的下一步就是进行板子的绘制以及焊接和组装并进行进一步的测试。 vscode使用技巧 在调试Onstep程序的过程中发现了一些小技巧，这里列举出来 全局搜索 Control + Shift + f 这个操作会在工作区搜索所有匹配的词条，并且可以使用正则匹配，这个是比较舒服的一点，在看Onstep程序的时候还是比较好用的，很轻松就可以找到想要的部分。 跳转到上一位置 这里是因为在调试代码的过程中总是要返回和前进，如果鼠标上没有前进和后退键，那就想要光标跳转到上一位置，可以使用快捷键 alt + ← 来实现跳转，同理，使用alt + →可以实现光标跳转到下一位置，这在代码调试中非常有用。 Vscode插件分享 在实际代码观看中发现一个比较严重问题，Onstep代码是使用Arduino IDE来进行编译和下载的，因此只有在Arduino IDE中才可以实现Control + 鼠标左键点击来实现跳转，这是比较伤的，因为这个IDE是很丑的。 模样是下面这个样子 界面我看着不舒服，不喜欢，而且很卡，因此就想办法来使用vscode进行编译以方便查看代码。 然后就找到了一个插件，叫做Arduino，可以对Onstep项目进行编译，方便跳转查看，这个插件安装和使用的教程我放在了链接中，这个链接的教程也是比较简单的。 设备 这周新添加了一个设备——23.8英寸的屏幕，因为笔记本的小屏幕太不舒服了，而且蓝牙键盘用不上，所以这周咬咬牙换了个大屏幕。 屏幕是KTC家的H24f8，定位是一款打游戏的屏幕，参数是1080p，180Hz以及23.8英寸，因为是最近新出的，所以首发价会便宜一点，488元拿下了，到手以后整体的效果还不错，因为是游戏屏幕，所以整个电脑的色彩算是比较丰富，也对比度也很高，另外这个屏幕官方也给到了软件驱动进行适配，并且频繁的更新固件，也算是非常不错的性价比，如果有需要的话，推荐入手。 学习 这周并没有额外的学习知识，属于是摆烂了一周，主要期末周没有考试就没有干劲，太捞了，但也是写了几个报告，并且将人工智能的实验做完，但实验报告还没做，不过这个实验也就是智能车调试，代码步骤跑通分析一下就可以，不需要自己写代码，算是比较简单的。 还有就是周末去参加了一次六级考试，感觉又是陪衬的一局，有点小可惜，浪费了报名费，不过再接再厉（如果这次过了，那就再也不考了，英语真是一生之敌）。 美食 这周值得纪念的美食有两次，第一次是在周六，第二次是在周日，都集中在了周末，也是比较巧合。 第一次是和同学约定一起吃的，一般每周都会和他去吃一次，算是一种默契吧，每周吃的东西不一样，这次吃的是包子。 这家店全称是——郭通天宫院小吃（马甸店），这家店的包子是比较好吃的，并且价格也可以接受，包子有3元，4元，5元的，和同学骑共享单车时间大概在20分钟左右，也是锻炼了一波身体，挺不错的。最主要是他里面的咸菜和腊八蒜是免费的，我很喜欢他们家的腊八蒜，美味。当然也会提供免费的小米粥、绿豆汤，只去吃包子的话，大概十来块就吃很饱了。 第二次是去老师家里面，这次是就业中心的老师请吃饭，如果没记错的话，上学期也请我吃饭了，算是一个每一个学期的固定项目，在老师家里吃的是火锅，和几个同事兼同学以及老师。 准备的食材还是比较丰富的，大家都吃的很饱，只有我吃到了最后，主要是锅里面还有剩的菜，我看到这些菜浪费了太可惜了，就咬咬牙全给造了，不完全是因为我很能吃，这顿完事以后我饱腹感特别强烈，一直到晚上都不是很饿，也就没有吃晚饭，晚上十点多才饿，就去小卖部买了一包辣条和一个面包填饱了肚子。 思考 前面也说了，这周我做了人工智能实验课的实验，主要是因为这周是教学周最后一周，而实验只能在实验室做，所以这周周五就是最后的期限来做实验了，时间还是比较紧张的。这里面有一个让我不太舒服的点。 那天下午我去做实验了，发现去的人很少，并且我们组的人就我一个到了，我是不相信都没看到群消息，还是有点小难受，后面组长也来了，和组长一起搭配然后将实验做完了，还好实验比较简单，两个人做也是绰绰有余。不过小组的事情让两个成员来做还是有点不舒服，问群里谁能来，最后只得到了一阵沉默，真操蛋。大学的小组作业变成个人作业的感觉真不好受，被白嫖，不过回过头来看的话，组里面成绩比我好的有好几个（一个小组六个人），他们肯定不会说放弃成绩，总是有一些野路子来完成实验，拿到好成绩，即使不去实验室。 总结 这周主要经历的事情就是上面的部分，事情有好有坏，内容偏叙述，下次争取整点硬核的技术内容，计算机怎么可能不涉及到技术是吧。 目标 上一周的目标基本完成，下一周要完成的内容如下，首先是赤道仪项目的跟进；然后是学习画电路图和PCB板子，最近看到同学在画赤道仪用到的板子才发现自己是一窍不通，是真垃圾；另外就是学习一下怎么绕过校园网认证，因为流量确实用的太快了，每个月都在给学校交钱，之前也考虑过这个问题，但是因为服务器没有ipv6的公网ip最终以失败告终，不过今天看到了一个视频，又激发了我的斗志，这一把我必须白嫖到校园网，如果成功，我专门出一期blog来讲讲实现，失败就不谈了。

前言 这是我的第一篇博客，以后会以一周一次的频率进行更新，主要内容是进行一周的总结，包括但不限于娱乐，比较好玩的事情，值得思考的东西，好用的软件，写的看的代码等等，是一个比较杂的博客，也并不会变成public，只是为了自己和朋友能看一看。如果有什么不准确的地方，请多多包涵，指出也是可以的，但是不合理的建议我会虚心接受，坚决不改的。 工作 博客搭建 首先说说博客搭建吧，我采用的方案是使用hugo+github pages来进行博客搭建。 hugo hugo官方是称自己是世界上用于构建网站最快的架构，并且给出了很多网站模版给用户使用，全部都是免费开源的模版，这点我是比较喜欢的。 对我搭建博客来说，它的功能就是将我的markdown文档转化为html文件，构建成为了静态网页的内容。 github pages github pages是github出的一个静态网页管理器，这个特性是非常适合用于管理hugo生成的静态网页的内容的，而且可以用户给出自定义域，那我的域名又可以用上了，非常舒服，touboli.site再次启动。 具体步骤 安装hugo，在github上建一个username.github.io的库，使用cloudflare将域名进行处理 hugo操作 hugo new site name 在当前目录中产生一个名字为name的文件夹，里面是hugo的相关文件夹。有些地方把这种操作成为创建站点。 cd name 进入name文件夹，可以进行进一步的工作了，文件夹的内容如下 其中主要用到的文件夹是content文件夹，写的文章都是放在这里content/post文件夹中；themes文件夹，使用的网页模版来自于themes文件夹，我主要使用hugo-PaperMod模版；public，存放hugo根据content中的文章生成的静态网页；hugo.toml，hugo的配置文件，官方文档在这里，可以进行自定义配置。 hugo new my-first-post.md 这会在content/posts 目录中创建该文件，在这个文件中初始会有title信息，date信息，draft信息，一般来说hugo在构建网站时不会发布草稿内容。 hugo 控制台输入hugo命令会转化你写的markdown文档为静态网页，生成的静态网页存放到public文件夹中。 hugo server -D 启动hugo的开发服务器来查看网站，在终端会显示URL。访问可以查看效果。 更多的有关内容可以查看hugo的文档 github pages操作 点击仓库的settings，找到pages部分，将DNS部分添加为域名即可 结合hugo + github pages 将hugo的public文件夹和创建的github pages关联起来，使用git完成 在public目录下执行以下命令 #在当前目录下新建代码库 git init #连接远程仓库 git remote add origin github的仓库链接 #查看本地所有分支 git branch #查看远程所有分支 git branch -a #切换分支到main分支 git checkout main #添加所有文件到暂存区 git add ....
